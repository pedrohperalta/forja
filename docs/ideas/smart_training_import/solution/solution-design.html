Problema

Cadastro manual de treinos é lento e propenso a erros

Toda vez que o treinador envia um novo programa de treinos (geralmente como fotos, uma por treino), o usuário precisa abrir o Forja e cadastrar manualmente cada exercício — digitando nomes, selecionando categorias, configurando séries, repetições e descanso. Um programa típico tem 2-3 treinos (A, B, C) com 5-7 exercícios cada. O processo leva ~30 minutos e resulta em erros frequentes de digitação, categorias erradas e detalhes trocados.

Digitação manual

15-20 exercícios × 6 campos cada = ~100 inputs manuais por troca de programa

Troca dolorosa

Sem forma de substituir o programa de uma vez — precisa deletar e recriar tudo

Objetivo

Importar um programa completo a partir de fotos em <5 minutos

Permitir que o usuário envie fotos dos treinos (uma por treino), processe via IA na nuvem, revise o resultado, e importe o programa completo no Forja — substituindo ou adicionando ao existente. O histórico de pesos e treinos anteriores é preservado.

<5min
Setup de programa completo
>85%
Precisão sem edição manual
~0

Solução Proposta

Arquitetura de alto nível

O app mobile captura fotos dos treinos, envia cada imagem para uma Supabase Edge Function que chama a API Claude Vision para extrair os exercícios. O resultado estruturado volta ao app, onde o usuário revisa e confirma a importação. Todo o processamento pesado acontece na nuvem; o app permanece offline-first para todas as outras funcionalidades.

Decisões técnicas

Supabase Edge Function

Escolhida por: API key segura no servidor, free tier generoso (500k invocações), base para futuro sync/web (DB + Auth + Storage + Realtime), open source (sem vendor lock-in forte), DX simples (supabase functions deploy).

Claude Vision (Sonnet 4.6)

Modelo multimodal que recebe a imagem e retorna JSON estruturado com exercícios extraídos. Sonnet 4.6 oferece boa relação custo/qualidade para OCR + interpretação semântica. Custo estimado: ~$0.02-0.05 por foto.

Offline-first preservado

A importação é a única funcionalidade que requer internet. Todas as outras features (treinos, histórico, planos) continuam 100% offline. Sem Supabase Auth ou DB no v1 — apenas Edge Functions.

Validação Zod Novo

// src/schemas/import.ts
import { z } from 'zod'

const extractedExerciseSchema = z.object({
  name: z.string().min(1),
  category: z.enum([...CATEGORIES]),
  equipment: z.string(),
  sets: z.number().int().min(1).max(20),
  reps: z.string().min(1),
  restSeconds: z.number().int().min(0).max(600),
  confidence: z.number().min(0).max(1),
})

const extractedWorkoutSchema = z.object({
  label: z.string().min(1),
  name: z.string().min(1),
  focus: z.string().min(1),
  exercises: z.array(extractedExerciseSchema).min(1),
})

export const extractWorkoutResponseSchema = z.object({
  workout: extractedWorkoutSchema,
})

A resposta da Edge Function é validada com Zod no client antes de exibir na tela de revisão. Se a validação falhar, trata como erro de processamento.

interface ImportState {
  photos: ImportPhotoStatus[]
  results: ExtractedWorkout[]
  mode: 'replace' | 'add'
  status: 'idle' | 'capturing' | 'processing' | 'reviewing' | 'done'

Edge Function — extract-workout

// supabase/functions/extract-workout/index.ts
import { serve } from 'https://deno.land/std/http/server.ts'
import Anthropic from 'npm:@anthropic-ai/sdk'

serve(async (req) => {
  const { image, label } = await req.json()

  const client = new Anthropic({
    apiKey: Deno.env.get('ANTHROPIC_API_KEY'),
  })

  const response = await client.messages.create({
    model: 'claude-sonnet-4-6',
    max_tokens: 2048,
    messages: [{
      role: 'user',
      content: [
        { type: 'image', source: {
          type: 'base64', media_type: 'image/jpeg', data: image
        }},
        { type: 'text', text: EXTRACTION_PROMPT(label) },
      ],
    }],
  })

  // Parse + validate JSON from Claude response
  // Return ExtractedWorkout or 422 error
})

A API key da Anthropic é armazenada como secret no Supabase (supabase secrets set ANTHROPIC_API_KEY=sk-...). Nunca exposta no client.

Client no app

// src/services/importApi.ts
const SUPABASE_URL = process.env.EXPO_PUBLIC_SUPABASE_URL
const SUPABASE_ANON_KEY = process.env.EXPO_PUBLIC_SUPABASE_ANON_KEY

export async function extractWorkout(
  imageBase64: string,
  label: string
): Promise<ExtractedWorkout> {
  const response = await fetch(
    `${SUPABASE_URL}/functions/v1/extract-workout`,
    {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'Authorization': `Bearer ${SUPABASE_ANON_KEY}`,
      },
      body: JSON.stringify({ image: imageBase64, label }),
    }
  )

  if (!response.ok) throw new ImportError(await response.json())

  const data = await response.json()
  return extractWorkoutResponseSchema.parse(data).workout
}

O anon key do Supabase é público por design — a segurança é via Row Level Security (não necessária aqui pois não usamos DB) e rate limiting. A API key da Anthropic fica protegida no servidor.

Dívida Técnica

Trade-offs aceitos

Sem autenticação Supabase

O anon key permite qualquer request à Edge Function. Aceitável para app pessoal. Em produção multi-usuário, adicionar Supabase Auth + RLS. Rate limiting via Supabase protege contra abuse básico.

Sem cache de resultados

Cada foto é processada do zero. Não há cache de resultados anteriores. Em v2, poderia cachear extrações por hash da imagem.

Sem suporte a PDF

Apenas fotos (JPEG/PNG) via expo-image-picker. PDF fica para v2 com expo-document-picker + conversão server-side.

Imagem como base64 no body

Imagens são enviadas como base64 no JSON body. Para fotos grandes, isso pode ser lento. Em v2, considerar upload para Supabase Storage + URL pré-assinada. Para v1, limitar resolução da imagem via ImagePicker (quality: 0.8, maxWidth: 1920).

Sem UI para planos arquivados

Planos arquivados ficam no MMKV mas não são acessíveis pela UI. Em v2, adicionar seção "Planos Anteriores" na tela de planos.

Impacto no Produto

O que muda para o usuário

Positivo

  • Setup de programa em <5 min vs ~30 min
  • Eliminação de erros de digitação
  • Troca de programa em uma operação (substituir tudo)
  • Preservação automática de histórico de pesos

Trade-offs

  • Requer internet para importação (primeira feature online do app)
  • Depende de foto legível — fotos ruins podem falhar
  • Custos de API (~$0.02-0.05/foto) — desprezível para uso pessoal

Sem mudança

  • Fluxo de treino (exercício → rest → checkpoint → complete)
  • Histórico e estatísticas
  • Criação manual de planos (continua disponível)
  • Funcionamento offline para todas as features existentes
