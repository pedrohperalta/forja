Visão Geral

Importação Inteligente de Treinos

O processo atual de cadastro de treinos no Forja é inteiramente manual: o usuário precisa digitar o nome de cada exercício, selecionar categorias, e configurar séries, repetições e descanso — um por um. Quando o treinador envia um novo programa (geralmente como PDF ou imagem), são necessários ~15-20 exercícios para configurar, o que leva tempo e é propenso a erros de digitação. Esta feature permite importar o programa automaticamente a partir de uma foto ou PDF, com um catálogo de exercícios integrado para ajustes rápidos.

Problema

Por que isso importa

Digitação manual de exercícios

Cada exercício exige digitar nome, equipamento e repetições manualmente. Com 15-20 exercícios por programa, erros de digitação são frequentes e difíceis de notar.

Configuração repetitiva de detalhes

Para cada exercício: selecionar categorias, definir séries, repetições e tempo de descanso. Multiplicado por ~18 exercícios, é um processo tedioso e desmotivante.

Ausência de catálogo de exercícios

Não existe uma base de exercícios pré-cadastrados. Toda vez que o usuário cria um exercício, parte do zero — mesmo para exercícios comuns como "Supino Reto" ou "Agachamento".

Troca de programa é dolorosa

Quando o treinador envia um novo programa, o usuário precisa deletar os planos antigos e recriar tudo do zero. Não há opção de importação ou substituição em lote.

Catálogo de Exercícios

Base de exercícios pré-carregada P0

Um banco de dados local com exercícios pré-cadastrados em pt-BR, organizados por grupo muscular e equipamento. Serve como base para o matching automático durante importação e como alternativa rápida ao cadastro manual.

Catálogo — busca + filtros por grupo muscular

Dados iniciais do catálogo

O catálogo será populado a partir de uma curadoria de ~200 exercícios comuns em academias brasileiras, organizada por:

  • Grupo muscular primário — alinhado com as 12 categorias existentes
  • Equipamento — Barra, Halter, Máquina, Cabo, Peso Corporal, etc.
  • Nome em pt-BR — com variantes e apelidos comuns para fuzzy match

Fonte potencial: adaptação do free-exercise-db (800+ exercícios, domínio público) traduzido para pt-BR, ou curadoria manual dos mais comuns.

Implementação

Fases

Fase 1 — Catálogo de Exercícios

P0
  • Tipo CatalogExercise e exerciseCatalogStore (Zustand + MMKV)
  • Seed data: JSON com ~200 exercícios em pt-BR (curadoria dos mais comuns)
  • Tela de catálogo: busca por nome + filtro por grupo muscular
  • Integração com formulário: "Buscar no Catálogo" como alternativa ao cadastro manual
  • Autocomplete: sugestões fuzzy no campo de nome do exercício

Fase 2 — Scan Inteligente com IA

P0
  • Edge function: proxy autenticado para Claude Vision API (Supabase Functions ou similar)
  • Prompt engineering: prompt otimizado para extrair planos + exercícios de imagens de treino
  • Tela de captura: câmera + seleção de galeria (reuso do expo-image-picker)
  • Tela de processamento: feedback visual durante análise
  • Matching automático: fuzzy match dos exercícios extraídos contra o catálogo

Fase 3 — Revisão e Importação

P0
  • Tela de revisão: visualização completa do programa extraído com indicadores de match
  • Edição inline: corrigir nomes, categorias, séries, reps, descanso antes de importar
  • Ação de substituição: substituir programa completo ou adicionar ao existente
  • Arquivamento: planos antigos arquivados (não deletados), preservando histórico
  • Migração de pesos: manter lastWeights para exercícios com mesmo nome

Fase 4 — Refinamento

P1
  • Feedback de erro: tratamento de falhas na extração (imagem ruim, formato inesperado)
  • Exercícios personalizados: salvar exercícios criados pelo usuário no catálogo
  • Melhorias de UX: animações de transição, estados de loading, acessibilidade
  • Testes E2E: fluxo completo com mocks da API de visão
Riscos e Mitigação

O que pode dar errado

RiscoSeveridadeMitigação
Qualidade da imagem/PDF — fotos com má iluminação ou texto cortado reduzem precisão Alta Guias visuais na câmera, suporte a múltiplas fotos, fallback para catálogo manual
Variação de formato entre treinadores — cada um tem um layout diferente Média Prompt engineering robusto com exemplos diversos. Tela de revisão garante validação antes de importar
Custo da API de visão — cada chamada tem custo, pode escalar com múltiplas tentativas Baixa Resize da imagem para ~1568px, cache por hash. Uso pessoal: ~1 importação/mês, custo marginal
Qualidade do catálogo em pt-BR — bases open-source são majoritariamente em inglês Média Curadoria manual dos ~200 exercícios mais comuns em academias brasileiras com nomes validados
Necessidade de backend — API key não pode ficar no app por segurança Média Edge function leve (Supabase Functions ou Cloudflare Workers) — proxy autenticado sem estado
Métricas de Sucesso

Como sabemos que funcionou

Tempo de configuração

Antes: ~30 min para cadastrar 3 planos com ~18 exercícios

Depois: <5 min com scan + revisão rápida

Taxa de erro de digitação

Antes: Erros frequentes em nomes, categorias incorretas

Depois: Nomes padronizados via catálogo, categorias automáticas

Precisão da extração por IA

Meta: >85% dos exercícios extraídos corretamente sem edição manual

Taxa de match no catálogo

Meta: >70% dos exercícios extraídos têm correspondência automática no catálogo

Fora do Escopo

O que NÃO está incluído

  • OCR on-device — extração feita na nuvem via Claude Vision, não localmente
  • Importação de texto puro / planilha — apenas imagem e PDF na v1
  • Sync cloud do catálogo — catálogo é local, sem sincronização entre dispositivos
  • Vídeos de demonstração de exercícios — apenas nomes e metadados no catálogo
  • Suporte a múltiplos idiomas no catálogo — apenas pt-BR na v1
  • Importação parcial — importa o programa completo, sem selecionar planos individuais
