Visão Geral

Configuração Dinâmica de Treinos

Atualmente o Forja possui 3 planos de treino fixos (A, B, C) com 21 exercícios pré-definidos em código. Esta feature permite que o usuário crie, edite e remova treinos completos — incluindo exercícios, séries, repetições, tempo de descanso, categoria e equipamento — diretamente no app, sem depender de atualizações de código.

Problema

Por que isso importa

Treinos são estáticos

Qualquer alteração nos treinos (trocar exercício, ajustar séries, adicionar um dia de treino) exige mudança no código-fonte e rebuild do app. Isso bloqueia o uso real no dia a dia.

Limite fixo de 3 treinos

O modelo atual suporta apenas 3 planos (A/B/C). Usuários com splits de 4, 5 ou 6 dias não conseguem usar o app como sua ferramenta principal.

Evolução ao longo do tempo

Treinos mudam frequentemente — a cada mesociclo, troca de foco ou avaliação com personal. O usuário precisa atualizar seus treinos de forma rápida e sem fricção.

Histórias de Usuário

O que o usuário precisa fazer

Como praticante de musculação

Quero criar meus próprios treinos com os exercícios que meu personal passou, para que o app reflita exatamente o que faço na academia.

Como praticante de musculação

Quero editar um treino existente (trocar exercício, mudar séries/reps, ajustar descanso) sem perder meu histórico de treinos anteriores.

Como praticante de musculação

Quero adicionar ou remover dias de treino livremente (ex: passar de ABC para ABCDE) conforme minha rotina muda.

Como praticante de musculação

Quero reordenar os exercícios dentro de um treino para refletir a sequência correta que faço na academia.

Como praticante de musculação

Quero configurar tempo de descanso por exercício, já que exercícios compostos precisam de mais descanso que isoladores.

Funcionalidades

Escopo da Feature

Criar Treino

P0

Criar um novo dia de treino com nome e foco muscular. Sem limite de quantidade. IDs gerados automaticamente em sequência (A, B, C, D...).

Editar Treino

P0

Alterar nome e foco de um treino existente. As mudanças afetam apenas treinos futuros — o histórico permanece imutável.

Remover Treino

P0

Excluir um dia de treino com confirmação em duas etapas (padrão Forja: APAGAR → CONFIRMAR). O histórico associado é preservado.

Adicionar Exercício

P0

Adicionar exercícios com nome digitado livremente (sem biblioteca fixa). Configurar séries, repetições, tempo de descanso, categoria e equipamento.

Editar Exercício

P0

Modificar qualquer parâmetro de um exercício existente: nome, séries, reps, descanso, categoria, equipamento.

Remover Exercício

P0

Remover um exercício de um treino. Confirmação em duas etapas. Fotos de equipamento associadas são preservadas.

Reordenar Exercícios

P1

Arrastar exercícios para reorganizar a ordem dentro de um treino. A ordem define a sequência durante a execução do treino.

Descanso por Exercício

P1

Configurar tempo de descanso individual por exercício (padrão: 60s). O rest timer usa esse valor em vez do fixo atual.

Foto de Equipamento

P2

Tirar ou trocar foto do equipamento diretamente na tela de configuração do exercício (reusa a feature existente).

Estado Vazio

P1

Quando não há treinos configurados, a home exibe mensagem convidando o usuário a criar seu primeiro treino, com link direto para a tela de configuração.

Nova rota top-level /plans acessível a partir de um botão na home. Dentro dela, duas sub-rotas:

  • /plans — Lista de todos os treinos com opção de criar novo
  • /plans/[id] — Detalhe/edição de um treino específico com lista de exercícios

A tela de adicionar/editar exercício pode ser um bottom sheet ou modal dentro da tela de detalhe do plano.

Critérios de Aceitação

Validação

Criar Treino

Dado que o usuário está na lista de treinos
Quando toca em "Novo Treino" e preenche nome e foco
Então um novo plano é criado com ID sequencial e aparece na lista
Dado que o usuário cria um treino
Quando volta para a home
Então o novo treino aparece como card disponível para iniciar

Editar Treino

Dado que o usuário edita o nome de um treino
Quando inicia um treino com esse plano
Então o novo nome aparece durante a execução
Dado que o usuário editou um treino
Quando visualiza o histórico de treinos anteriores
Então os treinos anteriores mantêm o nome original (histórico imutável)

Remover Treino

Dado que o usuário toca "Apagar Treino"
Quando confirma a exclusão (CONFIRMAR)
Então o treino é removido da lista e da home
Dado que o usuário removeu um treino com histórico
Quando acessa a tela de histórico
Então os treinos realizados com esse plano ainda aparecem normalmente

Gerenciar Exercícios

Dado que o usuário adiciona um exercício com nome, 3 séries, 12 reps, 90s descanso
Quando inicia o treino
Então o exercício aparece na fila com 3 séries e o rest timer usa 90 segundos
Dado que o usuário reordena exercícios no treino
Quando inicia o treino
Então a fila de exercícios segue a nova ordem

Estado Vazio

Dado que não há treinos configurados
Quando o usuário abre a home
Então vê mensagem de boas-vindas com CTA para configurar treinos

Migração

Dado que o usuário atualiza o app da versão com planos fixos
Quando abre o app pela primeira vez
Então os 3 planos ABC são migrados automaticamente para o planStore
Riscos e Mitigação

O que pode dar errado

RiscoSeveridadeMitigação
Perda de dados durante migração dos planos fixos Alta Manter constante original como fallback. Flag de migração no MMKV. Testes cobrindo cenário de migração.
Treino editado durante sessão ativa Média Bloquear edição de plano que está em execução (workoutStore.status === 'active'). Exibir aviso.
Histórico órfão após remoção de plano Baixa WorkoutSession já armazena planName/focus como snapshot. Não depende do plano existir.
Complexidade do drag-and-drop em React Native Média Usar react-native-draggable-flatlist (bem mantido, compatível com Reanimated). Fallback: botões ↑/↓.
IDs de exercício colidem entre planos Baixa Usar UUID v4 para ExerciseId (já é branded type). Colisão estatisticamente impossível.
Fora do Escopo

O que NÃO está incluído

  • Biblioteca de exercícios pré-definida — exercícios são criados manualmente pelo usuário
  • Templates compartilháveis — sem export/import de planos entre usuários
  • Programação periódica / mesociclos — sem agendamento automático de quando trocar treinos
  • Imagens de demonstração de exercícios — apenas fotos de equipamento (câmera do device)
  • Sync com backend / cloud — todos os dados permanecem locais no dispositivo
  • Duplicar treino — pode ser adicionado como melhoria futura
Notas Técnicas

Considerações de Implementação

Novo planStore

Criar src/stores/planStore.ts com Zustand + MMKV persistence. Seguir o mesmo padrão de workoutStore e appStore: atomic selectors, branded types, partialize para excluir estado efêmero.

Rotas

Adicionar src/app/plans/index.tsx e src/app/plans/[id].tsx como rotas top-level. Seguir padrão existente: arquivos de rota apenas em src/app/, lógica em components/hooks.

Home (index.tsx)

Substituir cards hardcoded por leitura do planStore. Adicionar botão de acesso à configuração. Implementar empty state quando plans.length === 0.

Rest Timer

O useRestTimer atualmente usa 60s fixo. Atualizar para receber restSeconds do exercício atual via workoutStore → planStore.

Imutabilidade do Histórico

O WorkoutSession já armazena planName e focus como snapshot no momento da conclusão. Nenhuma mudança necessária — o design atual já suporta histórico imutável.

Migração v1

Na inicialização do planStore, verificar flag plans_migrated no MMKV. Se falso, copiar planos de constants/plans.ts para o store e marcar flag. Manter a constante como fallback por 2 versões.
