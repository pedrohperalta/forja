Visao Geral

Registro Retroativo de Treino

Permitir que o usuario registre treinos realizados no passado de forma rapida, sem precisar passar pelo fluxo de treino em tempo real. O Forja permite que usuarios registrem treinos de academia em tempo real, acompanhando series, pesos e descanso. Porem, nem sempre o usuario consegue abrir o app durante o treino. Este documento especifica uma funcionalidade para registro simplificado de treinos passados, mantendo o historico completo e a progressao de cargas precisa.

Problema & Oportunidade

Por que isso importa

Historico incompleto

Quando o usuario esquece de abrir o app durante o treino, o historico fica com lacunas. Isso prejudica a visualizacao da evolucao e a confiabilidade dos dados de progressao.

Cargas desatualizadas

Sem o registro, as "ultimas cargas" ficam desatualizadas, fazendo o app sugerir pesos incorretos no proximo treino real.

Atrito do fluxo atual

O fluxo de treino em tempo real (serie por serie, com timer de descanso) e inapropriado para um registro rapido retroativo. O usuario quer apenas informar "fiz este plano neste dia com estes pesos".

Oportunidade

Ao permitir o registro retroativo, o Forja se torna a fonte unica e confiavel do historico de treinos do usuario. Isso aumenta o engajamento (menos frustracao com lacunas), melhora a precisao das sugestoes de carga e fortalece a retencao — o usuario passa a confiar no app como seu diario completo de treinos.

Publico-Alvo

Para quem e essa feature

Usuarios ativos do Forja que treinam regularmente e ocasionalmente esquecem de registrar o treino no app. Tipicamente lembram dos pesos utilizados e querem registrar dentro de uma semana.

Perfil Primario

Praticante de musculacao que treina 3-5x por semana. Usa o Forja como principal ferramenta de acompanhamento, mas eventualmente treina sem o celular ou esquece de abrir o app.

Motivacao

Manter o historico completo para acompanhar a evolucao de cargas e ter um registro confiavel de frequencia de treinos.

Premissas

Assumimos que o usuario:

  • Lembra dos pesos utilizados (pelo menos aproximadamente)
  • Registra o treino em ate 7 dias apos a realizacao
  • Ja possui planos de treino criados no app

Historias de Usuario

O que o usuario precisa fazer

P0 — Selecao de Plano

Como praticante de musculacao, quero selecionar qual plano de treino realizei, para que o registro retroativo use os exercicios corretos.

Given que o usuario abriu a tela de registro retroativo
When ele visualiza a lista de planos
Then deve ver todos os planos ativos (nao arquivados) com nome, foco e numero de exercicios
And ao tocar em um plano, ele e selecionado e os exercicios sao exibidos
P0 — Selecao de Data

Como praticante de musculacao, quero escolher a data em que realizei o treino, para que o historico registre o dia correto.

Given que o usuario selecionou um plano
When ele visualiza o seletor de data
Then deve ver os ultimos 30 dias como chips horizontais rolaveis
And o dia de ontem deve estar pre-selecionado
And o dia de hoje deve estar desabilitado (treino de hoje usa o fluxo normal)
P0 — Registro de Pesos

Como praticante de musculacao, quero informar os pesos usados em cada exercicio, para que minhas cargas fiquem atualizadas no historico.

Given que o usuario selecionou um plano
When ele visualiza a lista de exercicios
Then deve ver cada exercicio do plano com campo de peso pre-preenchido com a ultima carga registrada
And deve poder ajustar o numero de series (padrao: series prescritas pelo plano)
And exercicios com peso 0 serao ignorados no registro (considerados como nao realizados)
P1 — Duracao Opcional

Como praticante de musculacao, quero opcionalmente informar a duracao do treino, para manter o registro mais completo.

Given que o usuario esta registrando um treino retroativo
When ele visualiza o campo de duracao
Then o campo deve ser opcional, com placeholder "min"
And se nao preenchido, a duracao sera registrada como 0
P0 — Pontos de Entrada

Como praticante de musculacao, quero acessar o registro retroativo de forma facil a partir da home ou do historico.

Given que o usuario quer registrar um treino passado
When ele esta na tela de historico
Then deve haver um botao "+" visivel no cabecalho que leva a tela de registro
And na tela inicial (home), deve haver um link discreto "Registrar treino passado" abaixo dos cards de plano
P0 — Pre-Preenchimento Inteligente

Como praticante de musculacao, quero que os campos de peso venham pre-preenchidos com a ultima carga, para nao precisar digitar tudo do zero.

Given que o usuario ja realizou treinos anteriores com o plano selecionado
When os campos de peso sao exibidos
Then cada exercicio deve ter o campo de peso pre-preenchido com a ultima carga registrada
And se nao houver registro anterior, o campo deve iniciar vazio
P0 — Confirmacao e Salvamento

Como praticante de musculacao, quero confirmar e salvar o registro, atualizando meu historico e minhas ultimas cargas.

Given que o usuario preencheu plano, data e ao menos um peso
When ele pressiona "REGISTRAR"
Then o treino deve ser salvo no historico com a data selecionada
And as ultimas cargas devem ser atualizadas para cada exercicio registrado
And o usuario deve receber feedback haptico de sucesso
And deve ser redirecionado para a tela de historico

Anti-Requisitos

O que esta funcionalidade NAO faz

  • Nao permite editar treinos ja registrados no historico
  • Nao permite criar novos planos — so registra treinos de planos existentes
  • Nao importa dados de outros aplicativos ou formatos de arquivo
  • Nao substitui o fluxo de treino em tempo real — e um complemento para registros esquecidos
  • Nao inclui timer de descanso, contagem de series em tempo real, ou qualquer elemento do fluxo ativo
Erros & Casos Limites

Cenarios excepcionais

CenarioComportamento Esperado
Usuario nao tem planos criados Exibir mensagem orientando o usuario a criar um plano primeiro, com link para a tela de planos
Usuario nao preenche nenhum peso Botao "REGISTRAR" permanece desabilitado
Dois registros do mesmo plano na mesma data Permitir — o ID unico garante que nao haja conflito
Plano tem 15+ exercicios Lista rolavel com todos os exercicios, sem limite
Usuario tenta selecionar data futura Impossivel — o seletor mostra apenas datas passadas (hoje desabilitado)

Premissas & Validacao

Hipoteses a validar

PremissaValidacao
Usuarios lembram dos pesos usados em treinos recentes Pre-preencher com ultima carga reduz a carga cognitiva; monitorar % de campos alterados vs. aceitos
Registros retroativos ocorrem dentro de 7 dias Monitorar distribuicao de datas selecionadas; se >30% sao >7 dias, expandir janela
Funcionalidade sera usada raramente (1-2x/mes por usuario) Monitorar frequencia de uso; se alta, pode indicar problema com o fluxo principal
Riscos & Mitigacoes

O que pode dar errado

RiscoSeveridadeMitigacao
Dados imprecisos poluem o historico Media Pre-preencher com ultima carga (referencia), janela limitada a 30 dias.
Usuarios preferem o retroativo ao fluxo real Media O fluxo real oferece beneficios (timer, progressao automatica) que o retroativo nao tem. Monitorar proporcao de registros retroativos vs. normais.
Entrega

Fase Unica

Registro Retroativo Completo

  • Tela de registro com selecao de plano, data, pesos e duracao
  • Seletor de data com chips horizontais (ultimos 30 dias)
  • Pre-preenchimento de cargas a partir do historico
  • Salvamento no historico com atualizacao de ultimas cargas
  • Pontos de entrada na tela de historico e na home

Dependencia: Planos de treino ja criados pelo usuario.

Fora do Escopo

O que NAO esta incluido

  • Edicao de treinos ja salvos no historico
  • Registro detalhado serie por serie (peso diferente por serie)
  • Visualizacao de calendario completo com marcacoes
  • Importacao de dados de outros aplicativos
  • Registro de treinos avulsos (sem plano associado)
